Version explanation : x.yzwu
  x = development build
      0 -> uncomplete
      1 -> ready for public release
      2 -> ...
      3 -> ...

      y = number of features added to the game, when updated, "u" and "w" will reset

      w = number of major changes / bugfixes made in the current new feature, when updated, "u" will reset

      u = number of minor changes (does not affect gameplay) made during the current "x.yw" version
